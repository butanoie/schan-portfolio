# Code Review Skill

When reviewing code:
1. Check for security vulnerabilities (OWASP Top 10)
2. Look for performance issues (N+1 queries, memory leaks)
3. Verify error handling (edge cases, null checks)
4. Assess test coverage
5. **Review documentation and code quality using [CODE_REVIEW_GUIDELINES.md](../../docs/CODE_REVIEW_GUIDELINES.md)**

See [CODE_REVIEW_GUIDELINES.md](../../docs/CODE_REVIEW_GUIDELINES.md) for detailed documentation review standards and examples.
